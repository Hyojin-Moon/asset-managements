# 위험 요소 평가 및 롤백 전략

## 위험 요소 평가

식별하고 문서화할 항목:

### 기술적 위험
- API 변경
- 성능 이슈
- 네이티브 브릿지 호환성

### 의존성 위험
- 외부 라이브러리 업데이트
- 앱 버전별 차이

### 연동 위험
- iOS/Android 동작 차이
- 웹뷰 제한사항

## 위험 평가 테이블 양식

각 위험에 대해 다음을 명시합니다:

| 위험 요소 | 발생 확률 | 영향도 | 대응 방안 |
|----------|---------|--------|----------|
| [위험 설명] | 낮음/중간/높음 | 낮음/중간/높음 | [구체적인 조치 단계] |

### 확률/영향도 기준

**발생 확률**:
- 낮음: 거의 발생하지 않음 (< 20%)
- 중간: 발생할 수 있음 (20-60%)
- 높음: 발생할 가능성이 높음 (> 60%)

**영향도**:
- 낮음: 사용자 경험에 미미한 영향
- 중간: 일부 기능 제한 또는 우회 필요
- 높음: 핵심 기능 마비 또는 서비스 불가

## 롤백 전략

문제 발생 시 각 단계의 변경 사항을 되돌리는 방법을 문서화합니다.

### 고려 사항

1. **코드 변경 사항**
   - 되돌려야 할 파일 목록
   - git checkout 또는 revert 명령어

2. **설정 변경 사항**
   - 복원할 설정 파일
   - 환경 변수 변경 사항

3. **의존성**
   - 제거할 패키지
   - package.json 변경 사항

### 롤백 템플릿

```markdown
### N단계 실패 시

**복구 방법**:
1. 변경된 파일 되돌리기: [파일 목록]
2. git checkout으로 이전 상태 복원
3. [추가 정리 작업]

**확인 사항**:
- [ ] 이전 단계 기능 정상 동작
- [ ] 빌드 성공
- [ ] 테스트 통과
```

### Git 롤백 명령어

```bash
# 특정 파일 되돌리기
git checkout HEAD~1 -- path/to/file

# 특정 커밋으로 되돌리기 (새 커밋 생성)
git revert <commit-hash>

# 스테이지 취소
git reset HEAD path/to/file

# 변경사항 버리기 (주의: 복구 불가)
git checkout -- path/to/file
```
